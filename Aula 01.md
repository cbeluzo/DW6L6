## Aula 1: Preparação do Ambiente de Desenvolvimento


### Introdução ao Curso
Nesta aula inicial, damos as boas-vindas e compreendemos a importância do desenvolvimento web no cenário atual. Discutimos brevemente os objetivos do curso e como as habilidades que serão adquiridas são fundamentais para profissionais de Tecnologia da Informação.

#### Perguntas:
1. **Qual é a importância do desenvolvimento web no cenário atual?**
   - O desenvolvimento web desempenha um papel crucial, pois a maioria das interações e serviços acontecem através da web. É a base para aplicativos, sites e sistemas modernos.

2. **Quais são os objetivos principais deste curso?**
   - Este curso tem como objetivo apresentar os conceitos fundamentais, frameworks e práticas essenciais para o desenvolvimento de sistemas web com Java. Queremos capacitar os alunos a criar, implementar e manter sistemas web eficientes e escaláveis.

3. **Como as habilidades adquiridas neste curso podem beneficiar profissionais de Tecnologia da Informação?**
   - As habilidades adquiridas permitirão aos profissionais desenvolver aplicações web robustas, alinhadas com as melhores práticas do mercado. Isso abrirá oportunidades de trabalho em projetos desafiadores e relevantes.

### Instalação do JDK (Java Development Kit)
Em seguida, iniciaremos com a instalação do JDK, peça fundamental para o desenvolvimento em Java. Passo a passo, abordaremos a instalação do JDK nas plataformas mais comuns: Windows, macOS e Linux. Esse passo é crucial para garantir que todos estejam com o ambiente Java corretamente configurado.

#### Perguntas:
1. **Por que é necessário instalar o JDK para o desenvolvimento Java?**
   - O JDK é necessário porque fornece as ferramentas e bibliotecas essenciais para desenvolver e executar aplicativos Java. Inclui o compilador, bibliotecas padrão e outras utilidades.

2. **Quais são os passos para instalar o JDK no Windows?**

3. **Como garantir que a instalação do JDK foi bem-sucedida?**

### Solução

**Simples código para a criação de um projeto web básico utilizando Java, Servlets, e JSP. Este exemplo implementa a aplicação "Hello, World!".**

#### Servlet (HelloWorldServlet.java):
```java
import

 java.io.IOException;
import java.io.PrintWriter;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloWorldServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws IOException {
        response.setContentType("text/html");
        PrintWriter out = response.getWriter();
        out.println("<html><body>");
        out.println("<h2>Hello, World!</h2>");
        out.println("</body></html>");
    }
}
```

#### Página JSP (hello.jsp):
```jsp
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Hello, World!</title>
</head>
<body>
    <h2>Hello, World! This is from a JSP page.</h2>
</body>
</html>
```

**Para este exemplo:**
1. O Servlet `HelloWorldServlet` responde às solicitações GET e gera uma página HTML simples com a mensagem "Hello, World!".
2. A página JSP (`hello.jsp`) contém uma versão semelhante da mensagem "Hello, World!" e é chamada pelo Servlet.

### Atividade Prática 2: Desenvolvimento de uma Aplicação Web Simples
**Objetivo da Atividade:** Desenvolver uma aplicação web mais complexa utilizando Java, Servlets, JSP e incorporando conceitos de persistência de dados. Nesta atividade, os alunos criarão uma aplicação de lista de tarefas, permitindo adicionar, visualizar e excluir tarefas.

**Passos da Atividade:**
1. **Criação do Projeto:**
   - Crie um novo projeto web no Eclipse.
   - Configure a estrutura do projeto com pastas para Servlets, JSPs e classes de modelo.

2. **Modelo de Tarefa:**
   - Crie uma classe Java representando o modelo de uma tarefa (por exemplo, `Task`), com atributos como `id`, `description`, e `dueDate`.

3. **Persistência de Dados:**
   - Implemente uma classe para gerenciar a persistência de tarefas. Pode ser simples, utilizando uma lista em memória para armazenar as tarefas.

4. **Servlets:**
   - Crie Servlets para lidar com diferentes operações na aplicação, como adicionar uma tarefa, listar todas as tarefas e excluir uma tarefa.

5. **Páginas JSP:**
   - Desenvolva páginas JSP para apresentar a interface do usuário. Crie uma página para exibir a lista de tarefas, uma página para adicionar uma nova tarefa e uma página de confirmação para exclusão de tarefas.

6. **Integração Front-End e Back-End:**
   - Utilize formulários HTML nas páginas JSP para interação do usuário.
   - Implemente a lógica nos Servlets para processar as requisições dos formulários.

7. **Teste e Validação:**
   - Teste a aplicação localmente, utilizando o servidor Tomcat integrado.
   - Valide a adição, visualização e exclusão de tarefas, garantindo que os dados estão sendo manipulados corretamente.

**Entregáveis:**
   - Estrutura do projeto no Eclipse.
   - Código-fonte completo da aplicação, incluindo Servlets, JSPs e classes de modelo.
   - Capturas de tela demonstrando a execução da aplicação e suas funcionalidades.

**Avaliação:**
   - Os alunos serão avaliados com base na implementação correta da lógica do back-end (Servlets, modelo de tarefa, persistência de dados) e na integração eficiente com o front-end (páginas JSP e formulários). A funcionalidade e a usabilidade da aplicação também serão consideradas.

### Solução

**Vamos criar uma solução básica para a atividade proposta. Neste exemplo, utilizaremos apenas Java, Servlets e JSPs, sem a utilização de frameworks de persistência de dados mais avançados. Este é um ponto de partida, e em projetos reais, você poderia considerar a integração com um banco de dados e um framework ORM como o Hibernate.**

#### Criação do Projeto:
Crie um novo projeto web no Eclipse e configure a estrutura básica.

#### Modelo de Tarefa (Task.java):
```java
public class Task {
    private static int count = 1;
    private int id;
    private String description;
    private String dueDate;

    public Task(String description, String dueDate) {
        this.id = count++;
        this.description = description;
        this.dueDate = dueDate;
    }

    // Getters e Setters
}
```

#### Persistência de Dados (TaskManager.java):
```java
import java.util.ArrayList;
import java.util.List;

public class TaskManager {
    private static List<Task> tasks = new ArrayList<>();

    public static List<Task> getTasks() {
        return tasks;
    }

    public static void addTask(Task task) {
        tasks.add(task);
    }

    public static void deleteTask(int taskId) {
        tasks.removeIf(task -> task.getId() == taskId);
    }
}
```

#### Servlet para Adicionar Tarefa (AddTaskServlet.java):
```java
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/addTask")
public class AddTaskServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        String description = request.getParameter("description");
        String dueDate = request.getParameter("dueDate");

        Task newTask = new Task(description, dueDate);
        TaskManager.addTask(newTask);

        response.sendRedirect("taskList.jsp");
    }
}
```

#### Servlet para Listar Tarefas (TaskListServlet.java):
```java
import java.io.IOException;
import java.util.List;
import javax.servlet.RequestDispatcher;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/taskList")
public class TaskListServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        List<Task> tasks = TaskManager.getTasks();
        request.setAttribute("tasks", tasks);

        RequestDispatcher dispatcher = request.getRequestDispatcher("taskList.jsp");
        dispatcher.forward(request, response);
    }
}
```

#### Página JSP para Exibir Lista de Tarefas (taskList.jsp):
```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>Task List</title>
</head>
<body>
    <h2>Task List</h2>
    <ul>
        <c:forEach var="task" items="${tasks}">
            <li>${task.description} - Due Date: ${task.dueDate} <a href="deleteTask?id=${task.id}">Delete</a></li>
        </c:forEach>
    </ul>

    <h2>Add New Task</h2>
    <form action="addTask" method="post">
        Description: <input type="text" name="description" required><br>
        Due Date: <input type="text" name="dueDate" required><br>
        <input type="submit" value="Add Task">
    </form>
</body>
</html>
```

#### Servlet para Excluir Tarefa (DeleteTaskServlet.java):
```java
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/deleteTask")
public class DeleteTaskServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        int taskId = Integer.parseInt(request.getParameter("id"));
        TaskManager.deleteTask(taskId);

        response.sendRedirect("taskList.jsp");
    }
}
```
